# Usage Stats

Show the user their Claude Code usage stats for today with token breakdown and cost estimates. Output as a styled HTML report and open it in the browser.

**Announce at start:** "Checking your usage stats for today..."

## Data Sources

1. **`~/.claude/history.jsonl`** — one JSON object per line, each user prompt with `timestamp`, `project`, `sessionId`
2. **`~/.claude/stats-cache.json`** — historical aggregate stats with `dailyActivity`, `dailyModelTokens`, `modelUsage`, `totalSessions`, `totalMessages`
3. **`~/.claude/projects/*/<sessionId>.jsonl`** — conversation logs containing assistant messages with `message.usage` (input_tokens, output_tokens, cache_read_input_tokens, cache_creation_input_tokens) and `message.model`

## Steps

### 1. Run a single Python script

Run a single Python script that gathers all data, computes costs, generates a styled HTML file, and opens it in the browser.

**From history.jsonl** — filter to today's local date (midnight to now, timestamps are ms since epoch):
- Total prompts sent today
- Unique session IDs
- Prompts per project (use basename, show home dir as "~ (home)")
- Timestamp of first and last prompt

**From project .jsonl files** — for each of today's session IDs, walk `~/.claude/projects/` to find matching `<sessionId>.jsonl` files. Parse assistant messages where `message.usage` exists:
- Aggregate tokens per model: input_tokens, output_tokens, cache_read_input_tokens, cache_creation_input_tokens
- Count total API requests

**From stats-cache.json**:
- totalMessages, totalSessions
- dailyActivity for average/busiest day comparison

### 2. Compute estimated API cost

Use these rates (per million tokens) for cost estimation:

| Model | Input | Output | Cache Write | Cache Read |
|-------|-------|--------|-------------|------------|
| claude-opus-4-6 | $15 | $75 | $18.75 | $1.50 |
| claude-sonnet-4-6 | $3 | $15 | $3.75 | $0.30 |
| claude-sonnet-4-5-20250929 | $3 | $15 | $3.75 | $0.30 |
| claude-haiku-4-5-20251001 | $0.80 | $4 | $1 | $0.08 |

For any model containing "opus" use opus rates, "sonnet" use sonnet rates, "haiku" use haiku rates.

### 3. Generate HTML report

Write the report to `~/.claude/usage-report.html` and open it with `open` (macOS).

The HTML should be a single self-contained file (inline CSS, no external deps) with a dark theme dashboard look:

**Design requirements:**
- Dark background (#0d1117) with card-based layout
- Cards with subtle borders and rounded corners (#161b22 bg, #30363d borders)
- Accent color: bright green (#58a6ff for headings, #3fb950 for positive values)
- Cost values in orange/amber (#d29922)
- Clean sans-serif font (system-ui)
- Responsive layout with CSS grid (2 columns on desktop, 1 on mobile)

**Sections to include:**

1. **Header** — "Claude Code Daily Report" with date and time range, large and centered

2. **Stats row** — 4 stat cards in a row: Prompts, Sessions, API Calls, Est. Cost (large numbers, small labels underneath)

3. **Projects card** — horizontal bar chart showing prompts per project. Each bar is a colored div with width proportional to count. Show project name, count, and percentage.

4. **Tokens & Cost card** — for each model used:
   - Model name and pricing tier as a subheading
   - Table with columns: Category, Tokens, Cost
   - Rows: Input, Output, Cache Write, Cache Read
   - Horizontal colored bar next to each cost value (proportional to cost)
   - Bold total row at the bottom
   - Format tokens with commas, costs with $ and 2 decimal places

5. **Total tokens card** — big number showing total tokens used today with a breakdown donut/ring chart using CSS conic-gradient showing proportion of input/output/cache write/cache read

6. **Historical card** — all-time stats, daily average, today vs average comparison, busiest day. If stats-cache is stale, show a small warning badge.

**Additional polish:**
- Subtle box-shadow on cards
- Animate the bar widths on load with a CSS transition
- Use tabular-nums font-feature for number alignment
- Small footer: "Generated by /usage-stats skill · <timestamp>"

### 4. Summarize in terminal

After opening the browser, print a one-line summary in the terminal:
"Opened usage report in browser — <N> prompts, <N> tokens, $<cost> estimated today"

## Important

- Timestamps in history.jsonl are in milliseconds since epoch
- Use the local timezone for display
- If stats-cache.json is stale (lastComputedDate is old), note it in the Historical card
- If no prompts found for today, show an empty state message
- The script must be fully self-contained — no npm/pip dependencies beyond Python stdlib
- Always overwrite the same file (~/.claude/usage-report.html) so it doesn't create clutter
